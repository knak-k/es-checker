# ES添削ツール — 開発プラン

就活のES（エントリーシート）を、書いた直後にAIと自動チェックで添削できるWebアプリ。
自分が毎日使い、かつ「AIを開発に組み込む」志望の軸を体現するポートフォリオにする。

最終更新：2026-07-07

---

## 目的・価値

- **実需**：ESを書く→その場で添削→直す、を高速で回す（自分が毎日使う）
- **ポートフォリオ**：AI（Claude API）を実務プロセスに組み込んだ実物として面接で語れる
- **既存資産の活用**：就活CLAUDE.mdの「ES採点ルーブリック」「減点条件」「御社/貴社ルール」をそのまま移植

## 想定ユーザー・使い方

1. 設問・回答本文・企業名・職種・文字数上限を入力する
2. その場で「文字数（改行/空白除外）」「御社⇄貴社の誤用」を即チェック（AI不要・一瞬）
3. 「添削する」でAI（Claude）が採点＋改善案を返す
4. 直して再チェック、を繰り返す

## MVP（最初に作る最小構成）

- 入力フォーム（設問／回答／企業名／職種／文字数上限）
- **即時チェック（クライアント完結）**：文字数カウント、御社/貴社チェック
- **AI添削（サーバー経由でClaude呼び出し）**：5項目採点（各20点）＋減点チェック＋総評＋優先改善3つ
- 結果表示（項目ごとに 得点／理由／不足／改善案）

MVPは**保存なし＝貼って添削するだけ**。ログイン・DBは不要。

## 機能ロードマップ（優先度）

- **P0（MVP）**：上記
- **P1**：localStorage に下書き保存、企業別に管理、履歴
- **P2**：ESパーツ（使い回し部品）テンプレ、字数別テンプレ、バージョン比較
- **P3**：面接想定問答版（話し言葉なので「御社」チェックに切替）、テキスト書き出し

## AI添削の設計

- 採点軸（各20点・計100）：**主体性／思考力／実行力／他者理解力／再現性**（就活CLAUDE.md 8章を移植）
- 各項目で「得点・評価理由・不足要素・具体的改善案」を返す
- 減点チェック：数字偏重／感情語での価値補強／抽象語（信頼・成長・挑戦）の未定義使用／善意のみで仕組み設計欠如
- 既存ルール準拠チェック：結論ファースト／受け身表現の有無／目的の具体性／2点構造の回避／まとめ一文／AIっぽい日本語／企業独自性
- **辛口評価**（初稿で80点超は稀）、出力フォーマットは固定
- 応募職種に応じた視点（研究開発職／SIer等で評価軸が変わる）を指示に含める

## 技術スタック

- Next.js 16 + TypeScript + Vercel（あひるの英語と同構成で立ち上げ最速）
- AI は **サーバールート `/api/review`** から Anthropic SDK（Claude）を呼ぶ
- ⚠️ **APIキーはサーバー側の環境変数のみ**。クライアントに絶対出さない
- 即時チェック（文字数・御社/貴社）はクライアントで完結（APIコスト0）

## データ（MVP）

```ts
type ES = {
  id: string;
  company: string;   // 企業名
  role: string;      // 応募職種
  question: string;  // 設問
  body: string;      // 回答本文
  maxChars: number;  // 文字数上限
  updatedAt: string;
};
```

MVP は貼り付け→添削のみ（永続化なし）。P1 で localStorage 保存。

## コスト・注意

- Claude API は従量課金。キーは Vercel の環境変数に設定する
- 添削は「参考」であり最終判断は本人（AIの誤りもある旨を画面に明示）
- 文字数カウントは必ずプログラムで計算（`len(改行・空白を除いた本文)`）

## 開発ステップ

1. Next.js を scaffold ＋ 入力フォーム＋即時チェック（文字数・御社/貴社）をAIなしで実装
2. `/api/review` に Claude 添削を実装し、結果表示をつなぐ
3. localStorage 保存・企業別管理
4. パーツ/テンプレ、面接版
